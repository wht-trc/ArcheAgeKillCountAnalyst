# ArcheAge Kill-Count Analyst: AI-ассистент для аналитики PvP-праймов

## Описание проекта
Проект представляет собой специализированного AI-агента для автоматического анализа статистики PvP-праймов в MMORPG ArcheAge.  
Ассистент ориентирован на активных игроков, которые ведут статистику убийств в Google-таблицах и нуждаются в быстрых ответах на вопросы о результатах.

**Цель агента** — избавить игроков от ручного поиска, фильтрации и расчетов, предоставив удобный интерфейс с возможностью задавать вопросы на естественном языке и мгновенно получать точную статистику. Статистика PvP-праймов ведётся в Google Таблицах, разбитых по месяцам. Ручной поиск нужного игрока, подсчет средних значений, составление топов требуют времени и навыков работы с формулами.

Реализован **Agentic Workflow (агентный подход)** с одним инструментом, который строго следует правилам:
- Агент **не придумывает цифры** — все ответы формируются только на основе данных из Google Таблиц.
- Реализован **гибкий поиск** игроков: точный, префиксный и нечёткий, с учётом кириллицы и латиницы.
- Системный промпт агента содержит **инструкции, примеры и формат ответа**, что минимизирует риск неверной интерпретации вопросов.

## Технологический стек
- **Оркестрация:** LangChain.
- **Языковая модель:** DeepSeek Chat API.
- **Внешний источник данных:** Google Sheets API (gspread, сервисный аккаунт).
- **Инструмент агента:** кастомная функция get_kill_stats с прямым чтением листов и аналитикой (средние, суммы, топы, история ников).
- **Интерактивный интерфейс:** Gradio (чат с историей диалога и отчётом о выполнении каждого запроса).
- **Мониторинг:** кастомный класс `AgentMonitor` (время выполнения, число вызовов LLM и инструмента).

## Архитектура агента
Агент построен по схеме **один агент — один инструмент**:
1. **Пользователь задаёт вопрос** на естественном языке (например, «Кто топ-1 за все время?»).
2. **Системный промпт** описывает роль, контекст, правила и примеры вызова инструмента.
3. **Агент анализирует вопрос** и, если требуется, вызывает `get_kill_stats` с нужными параметрами (`action`, `metric`, `period` и т.д.).
4. **Инструмент** загружает листы таблицы, парсит строки, извлекает данные, фильтрует по дате/серверу/игроку, выполняет вычисления (среднее, сумма, эффективность) и возвращает текстовый результат.
5. **Агент формулирует ответ**, добавляя краткий аналитический комментарий (если уместно).

Особенности реализации:
- Парсинг CSV адаптирован под реальную структуру таблиц (пустые столбцы, BOM, кириллица).
- Игроки идентифицируются по **постоянному нику** (в скобках), который нормализуется (нижний регистр) для объединения разных написаний.
- При отсутствии точного совпадения по нику инструмент предлагает похожие варианты (префиксный, затем нечёткий поиск).
- Для вопросов «кто лучший» по умолчанию используется **среднее значение** (чтобы новички не проигрывали ветеранам по сумме), если явно не указано «суммарные».

## Мониторинг
Класс `AgentMonitor` собирает метрики в реальном времени:
- **Execution Time** — общее время выполнения запроса.
- **LLM Calls** — количество циклов рассуждения языковой модели.
- **Tool Calls** — количество обращений к инструменту.

Отчёт выводится в консоль Colab и дублируется в интерфейсе Gradio.

##  Как запустить в Google Colab
1. **Откройте ноутбук** в Google Colab.
2. **Добавьте секреты**: `DEEPSEEK_API_KEY` со своим API-ключом DeepSeek, `GCP_SERVICE_ACCOUNT` – всё содержимое JSON-файла сервисного аккаунта Google Cloud.
3. **Запустите все ячейки** по порядку.
4. **Откройте Gradio-интерфейс** по появившейся ссылке.

## Примеры вопросов
- «Выведи данные за 07.06.2026»
- «Кто топ-1 за все время?»
- «Расскажи о персонаже Olven»

## Результаты и выводы
- Агент успешно извлекает данные из таблицы, обрабатывает сложные структуры с пустыми колонками и примечаниями.
- Использование низкой температуры (0.1) гарантирует точность ответов и предотвращает галлюцинации.
- Гибкий поиск позволяет находить игроков по любому фрагменту ника, независимо от языка.
- Мониторинг позволяет оценить производительность и отладить работу агента.
